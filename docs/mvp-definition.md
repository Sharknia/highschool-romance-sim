# Heroine-First VN Maker MVP

## 한 줄 정의

히로인 라이브러리에서 히로인을 만들고, 프로젝트에는 히로인 1명과 루트 1개만 배정한 뒤, 자연어 이벤트를 씬/선택지/CG 생성 패치로 바꾸고, 검증과 승인 후 실제 플레이 가능한 웹 빌드로 export하는 로컬 AI 미연시 제작툴.

## MVP의 핵심 가설

이 제품의 첫 검증 대상은 "AI가 미연시 전체를 자동으로 만든다"가 아니다.

검증할 가설은 다음 하나다.

> 사용자가 자연어로 짧은 이벤트를 입력하면, 시스템이 이를 검증 가능한 VN 프로젝트 패치로 바꾸고, 이미지 생성까지 포함한 결과를 실제 플레이 가능한 빌드로 내보낼 수 있다.

## 단계 구분

이 문서는 MVP를 Alpha와 Beta로 나눈다.

- Alpha: 가장 먼저 구현할 뼈대. 제품 가설 검증에 직접 필요한 기능만 포함한다.
- Beta: Alpha가 성공한 뒤 제작툴로서 반복 사용성을 올리는 기능을 포함한다.

## Alpha

### Alpha 목표

히로인 라이브러리에서 히로인을 만들고, 프로젝트에는 히로인 1명만 선택한다. 선택된 히로인의 단일 루트에서 자연어 이벤트를 입력하면, 씬/선택지/CG 이미지 생성 작업이 구조화 패치로 제안되고, 사용자가 승인하면 프로젝트에 적용된다. 이후 플레이 프리뷰와 export 결과물에서 해당 이벤트를 실제로 플레이할 수 있어야 한다.

### Alpha 범위

- 단순 히로인 라이브러리
- 히로인 추가/수정/삭제
- 프로젝트당 히로인 1명
- 프로젝트 1개
- 루트 1개
- 자연어 이벤트 입력
- 씬/선택지/CG 패치 생성
- CG 실제 이미지 생성
- 패치 검증
- 패치 승인/취소
- SQLite 프로젝트 저장
- 플레이 프리뷰
- 웹 플레이어 export

### Alpha 사용자 흐름

1. 사용자가 히로인 라이브러리에서 히로인을 만든다.
2. 히로인의 이름, 설명, 성격, 말투, 외형 설명을 입력한다.
3. 사용자가 히로인의 기본 포트레이트를 지정하거나 생성한다.
4. 사용자가 새 프로젝트를 만든다.
5. 프로젝트에 사용할 히로인 1명을 선택한다.
6. 선택한 히로인이 프로젝트 내부 스냅샷으로 복사된다.
7. 시스템이 선택 히로인의 단일 루트를 만든다.
8. 사용자가 루트 화면에서 삽입 위치를 선택한다.
9. 사용자가 자연어 이벤트를 입력한다.
10. AI가 이벤트를 분석해 씬 수, 선택지, CG 필요 여부를 판단한다.
11. AI가 구조화된 프로젝트 패치를 제안한다.
12. 시스템이 패치를 복사본에 적용해 검증한다.
13. 사용자가 패치 요약, diff, 검증 결과를 확인한다.
14. 사용자가 승인하면 패치가 프로젝트에 적용된다.
15. CG가 필요한 경우 이미지 생성을 실행하고 결과를 프로젝트 에셋에 연결한다.
16. 사용자가 플레이 프리뷰에서 이벤트를 확인한다.
17. 사용자가 프로젝트를 export한다.
18. export된 `index.html`에서 루트를 플레이할 수 있다.

### Alpha 히로인 라이브러리

Alpha에는 단순 히로인 라이브러리를 포함한다.

포함 기능:

- 히로인 목록
- 히로인 추가
- 히로인 수정
- 히로인 삭제
- 기본 포트레이트 생성/교체

필수 필드:

- 이름
- 설명
- 성격
- 말투
- 외형 설명
- 기본 포트레이트

Alpha에서 히로인은 여러 명 만들 수 있다. 다만 프로젝트에는 히로인 1명만 선택할 수 있다.

프로젝트에 히로인을 선택하면 라이브러리 원본을 직접 참조하지 않고 프로젝트 내부 스냅샷으로 복사한다. 이후 라이브러리 원본을 수정해도 기존 프로젝트 히로인은 자동 변경되지 않는다.

삭제 정책은 단순하게 유지한다.

- 프로젝트에서 사용 중인 히로인을 라이브러리에서 삭제해도 기존 프로젝트 스냅샷은 유지한다.
- 삭제는 라이브러리 목록에서만 제거한다.
- 프로젝트 스냅샷 자동 동기화와 삭제 복구는 Alpha 범위가 아니다.

Alpha에서 제외하는 히로인 관리:

- 검색
- 정렬
- 폴더
- 태그 관리
- 히로인 복제
- 라이브러리와 프로젝트 스냅샷 자동 동기화
- 기본 감정 에셋 5종 자동 생성
- 추가 태그 에셋 생성

### Alpha 이미지 생성

이미지 생성은 Alpha에 포함한다.

Alpha에서 필요한 이미지 생성은 다음 두 가지다.

1. 히로인 기본 포트레이트 생성 또는 지정
2. 자연어 이벤트 패치에서 필요한 CG 1개 생성

Alpha에서는 기본 감정 에셋 5종 자동 생성과 추가 태그 에셋 생성을 하지 않는다.

즉, Alpha는 "이미지 생성이 되는 제작 루프"를 검증하되, 감정별 에셋 세트 관리까지는 하지 않는다.

### Alpha 자연어 이벤트 예시

입력:

```txt
하루와 도서관에서 있던 일이야.
하루가 책을 떨어트리고, 내가 책을 주워주려다가 두 사람의 손이 겹쳐.
둘 다 당황해서 어색해지는 짧은 러브코미디 이벤트로 만들어줘.
씬은 3개, CG는 1개만 만들어줘.
```

AI 판단:

```json
{
  "sceneCount": 3,
  "choiceCount": 1,
  "cgCount": 1,
  "newExpressionAssetCount": 0,
  "tone": "romantic_comedy"
}
```

패치 결과:

- 씬 3개 추가
- 필요한 선택지 1개 추가
- 기존 씬과 새 씬 연결
- CG asset 추가
- CG generation job 추가
- CG 실제 이미지 생성 후 asset에 연결

### Alpha 패치 계약

LLM은 전체 프로젝트를 다시 쓰면 안 된다. Alpha에서는 작은 이벤트 패치만 생성한다.

요청:

```ts
interface EventExpansionRequest {
  projectDirectory: string;
  baseProjectHash: string;
  routeId: string;
  afterSceneId: string;
  heroineId: string;
  userEvent: string;
  heroineContext: {
    name: string;
    description: string;
    personality: string;
    speechStyle: string;
    appearance: string;
  };
  constraints: {
    maxScenes: number;
    maxChoices: number;
    maxCgCount: number;
    allowNewExpressionAssets: false;
    language: "ko";
    contentRating: "all" | "teen";
  };
}
```

응답:

```ts
interface EventExpansionPlan {
  summary: string;
  decision: {
    sceneCount: number;
    choiceCount: number;
    cgCount: number;
    newExpressionAssetCount: 0;
  };
  patch: VnMakerProjectPatch;
}
```

허용 연산:

- `addScene`
- `updateScene`
- `updateSceneLink`
- `addChoice`
- `addAsset`
- `addGenerationJob`

금지 연산:

- 프로젝트 전체 재작성
- 루트 전체 재작성
- 씬 삭제
- 프로젝트 히로인 삭제
- 새 히로인 추가
- 새 루트 추가
- 호감도/스탯 시스템 추가
- 검증 없는 자동 적용

### Alpha 검증 조건

패치 적용 전 반드시 검증한다.

- 프로젝트에는 히로인 1명만 존재해야 한다.
- 프로젝트에는 루트 1개만 존재해야 한다.
- 모든 씬 `next`가 존재해야 한다.
- 모든 선택지 `next`가 존재해야 한다.
- 씬에 등장하는 캐릭터는 프로젝트 히로인과 일치해야 한다.
- `cgAssetId`가 있으면 asset이 존재해야 한다.
- CG asset은 generation job과 연결되어야 한다.
- generation job의 `outputAssetId`는 asset과 연결되어야 한다.
- 새 감정 에셋은 생성하지 않아야 한다.
- 패치가 선언한 `sceneCount`, `choiceCount`, `cgCount`와 실제 변경량이 일치해야 한다.
- 검증 실패 시 프로젝트 원본을 유지해야 한다.

### Alpha 플레이 프리뷰

필수 조건:

- 첫 씬부터 플레이 가능
- 현재 선택 씬부터 플레이 가능
- 히로인 기본 포트레이트 표시
- CG 표시
- 선택지 클릭 시 다음 씬 이동
- 검증 실패 프로젝트는 프리뷰 실행 전 경고 표시

### Alpha Export

Alpha의 export는 반드시 실제로 플레이 가능한 결과물을 만든다.

산출물 예:

```txt
exports/<project-id>/
  index.html
  assets/
    generated/
    source/
  runtime/
  project-data.json
```

필수 기능:

- `VnMakerProject`를 플레이어 런타임 데이터로 변환
- 플레이어 HTML 생성
- 런타임 JS 포함
- 프로젝트 데이터 포함
- 필요한 에셋 복사
- 에셋 경로 재작성
- export 전 검증
- 검증 실패 시 export 차단
- export 후 smoke test

export smoke test:

- `index.html` 로드 가능
- 첫 씬 표시
- 히로인 기본 포트레이트 표시
- 선택지 표시
- 선택지 클릭 후 다음 씬 이동
- CG가 있는 씬에서 CG 경로가 깨지지 않음

### Alpha 성공 기준

아래 시나리오가 끝까지 동작하면 Alpha 성공이다.

1. 새 프로젝트를 만든다.
2. 히로인 라이브러리에서 히로인 `하루`를 만든다.
3. 하루의 설명, 성격, 말투, 외형을 입력한다.
4. 하루 기본 포트레이트를 생성한다.
5. 프로젝트에 하루를 선택한다.
6. 하루가 프로젝트 스냅샷으로 복사된다.
7. 하루 루트가 자동 생성된다.
8. 루트 화면에서 도서관 이벤트를 자연어로 입력한다.
9. AI가 씬 3개, 선택지 1개, CG 1개로 판단한다.
10. 패치 diff와 검증 결과가 표시된다.
11. 사용자가 승인한다.
12. 패치가 프로젝트에 적용되고 SQLite에 저장된다.
13. CG 이미지가 생성되고 프로젝트 asset에 연결된다.
14. 플레이 프리뷰에서 추가된 이벤트를 확인한다.
15. 프로젝트를 export한다.
16. export된 `index.html`에서 루트를 플레이할 수 있다.

## Beta

### Beta 목표

Alpha에서 검증한 자연어 패치와 export 흐름을 제작자가 반복적으로 쓰기 좋게 만든다.

### Beta 범위

- 히로인 재사용 이력 표시
- 프로젝트 스냅샷 출처 표시
- 히로인 복제
- 히로인 검색/정렬/태그
- 기본 감정 에셋 5종 자동 생성
- 추가 태그 에셋 생성
- 이미지 생성 일괄 워크플로
- 에셋 보드
- 패치 히스토리
- 되돌리기
- 더 나은 씬 편집 UI
- 더 나은 route/scene 목록 UI

### Beta 히로인 라이브러리 개선

Beta에서는 Alpha의 단순 히로인 라이브러리를 반복 제작에 맞게 개선한다.

추가 기능:

- 히로인 복제
- 검색
- 정렬
- 태그
- 프로젝트별 사용 여부 표시

자동 동기화는 Beta에서도 필수는 아니다. 필요하면 후속 단계로 둔다.

### Beta 기본 감정 에셋

Beta에서 기본 감정 에셋 5종 자동 생성을 추가한다.

기본 태그:

- `normal`
- `happy`
- `sad`
- `angry`
- `shy`

### Beta 추가 태그 에셋

유저가 직접 태그를 입력해 추가 에셋을 생성할 수 있다.

예:

- `embarrassed`
- `library_smile`
- `worried_uniform`
- `festival_nervous`

### Beta 이미지 생성 일괄 워크플로

Beta에서는 이미지 생성 작업을 하나씩 실행하는 수준을 넘어, 여러 generation job을 관리하는 보드를 제공한다.

필수 기능:

- planned/running/completed/failed 상태 표시
- 여러 이미지 작업 선택 실행
- 실패 작업 재시도
- 생성 결과 미리보기
- asset 교체
- prompt/provider 메타데이터 표시

### Beta UX 개선

Beta에서 개선할 화면:

- 히로인 보드
- 프로젝트 보드
- 루트/씬 목록
- 씬 편집기
- 자연어 패치 diff
- 에셋 보드
- 플레이 프리뷰 패널
- export 결과 확인 패널

## MVP에서 삭제하는 범위

Alpha와 Beta 모두에서 제외한다.

- 프로젝트당 히로인 여러 명
- 복수 루트
- 여러 히로인 간 교차 분기
- 복잡한 호감도/스탯 시스템
- 오디오/보이스
- Live2D
- 팀 협업
- 마켓플레이스
- 자동 퍼블리싱
- Electron/Tauri 패키징
- 여러 LLM provider 설정 UI
- 전체 게임 자동 생성

## 콘텐츠 안전 기준

MVP 기본 콘텐츠 등급은 `teen` 이하로 둔다.

특히 학교 배경, 고등학생 설정, 미성년자로 해석될 수 있는 캐릭터가 있을 경우 선정적 CG 생성은 차단한다. 자연어 입력이 신체 접촉이나 선정적 묘사를 포함하더라도, 출력은 로맨틱 코미디 수준의 당황, 어색함, 감정 변화 중심으로 완화한다.

## 최종 정의

Alpha는 가장 작은 end-to-end 검증이다.

> 히로인 라이브러리에서 히로인을 만들고, 프로젝트에는 히로인 1명만 선택한 뒤, 자연어 이벤트를 씬/선택지/CG 생성 패치로 바꾸고, 승인 후 적용해, 실제 플레이 가능한 웹 빌드로 export한다.

Beta는 반복 제작에 필요한 사용성을 추가한다.

> 히로인 라이브러리, 감정 에셋 관리, 이미지 생성 작업 보드, 패치 히스토리를 더해 제작자가 같은 흐름을 여러 번 안정적으로 반복할 수 있게 만든다.
